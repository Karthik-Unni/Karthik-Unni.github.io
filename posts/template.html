What I learned in month one at Magma

January 2025 · 5 min read

Start with the most important thing — what is this post about and why did you write it? One or two sentences. Don't overthink it.

## background

What context does the reader need? Where were you before this? What problem were you trying to solve?

## what I learned

The actual content. Write like you're explaining it to a friend. Use simple language. If you're summarizing a paper, describe the core idea in your own words.

### key idea 1

Explain it. If there's code, show it: 

# example code block
def something():
    return "your code here"

### key idea 2

Another concept. Use lists if it helps:

  • thing one
  • thing two — inline code looks like this
  • thing three

## what I built

Did you build something from this? Describe it. Link to the GitHub repo if it's public. Even a small experiment counts.

## what surprised me

The part you didn't expect. The thing that clicked. The thing that's still confusing. This is often the most useful section for people reading later.

Optional: a quote or insight worth calling out on its own.

## what's next

What are you going to do with this? What do you want to learn next? What questions does this open up?
